      办公环境下如何同时连接内外网

      somenzz 2020-11-16 同时内外网

      办公室环境下，一般有两种网络，一个是内网，一个是外网，内网用于公司内部的办公网络，如 OA，人力资源管理系统，内部通讯软件等，外网一般用于查阅相关资料。我见过的场景是通常是两台电脑，一台用于外网，一台用于内网，互不干扰。

      不涉及内外网传输时没有什么痛点，但是对于开发人员来讲，几乎是不可能的：你从外网下载一段程序，除非你想亲自抄写一遍，否则，你需要把它传入内网环境。一般的，会想到用一个 “smart link”的软件，将两台电脑用 USB 线连接，这样使用一台鼠标就可以操作两台电脑，可以共享粘贴板和复制文件，少数人则使用 VPN 同时连接内外网，即使是 VPN，有些情况下，内外网直接的复制和粘贴依然是禁止的。

      往往公司会禁用 USB 接口，防止病毒或者内网文件泄露等。上述方法便不可行。其实还有一种方法，就是使用双网卡，一个连接内网，一个连接外网，比如常见的，大家都用笔记本电脑工作，可以使用无线网卡连接外网，有线网卡连接内网，如果可以同时连接内外网，则可以大大提升工作效率，公司明令禁止的另当别论。本文就是介绍这种同时连接内外网的方法，共分三步，非常简单，小白也可以操作。

      # 步骤一、设置网卡优先级

      通常外网的网段无法穷举，内网却可以，比如“192.168.0.0”，因此把连接外网的网卡设置为优先，也就是说通常情况下我们是访问外网的，某些特殊的网段，如“192.168.0.0”可以通过设置路由走内网。

      # 添加路由

      windows 以管理员权限打开 cmd:

      执行：

      route add 192.168.0.0 mast 255.255.0.0 10.1.46.1
      1

      linux 或 mac 则执行：

      sudo route add -net 192.168.0.0 10.1.46.1
      1

      也可以为某个 ip 单独添加路由：

      #win
route add 192.168.219.130 mask 255.255.255.255 10.1.46.1
#linux or mac
sudo route add -net 192.168.219.130 10.1.46.1
      1
      2
      3
      4

      如果要删除，把 add 改成 delete 即可。注意如果切换了网络环境，或者重启电脑，可能需要重新执行添加路由这一步，因此建议你写成脚本每次执行下脚本即可。你也可以想办法添加到静态路由表中，不建议新手这么做。

      至此你可以通过 ip 192.168.219.130 地址访问 xyz.abc.com 。

      # 步骤三、添加主机域名

      不可能一直通过 ip 来访问，如果 ip 多了根本记不住，也不方便，因此我们需要修改 hosts 文件帮我们自动记忆 ip 和域名的映射关系。

      因此 hosts 文件末尾中添加一条 IP 和对应的域名即可：

      192.168.219.130 xyz.abc.com
      1

      windows 的 hosts 文件位于c:\Windows\System32\drivers\etc\ ，linux 或 mac 位于 /etc/hosts。windows 无法直接编辑，可以复制出来修改完后再覆盖即可，修改之前请备份之前的 hosts 文件，万一不小心被清空可以恢复。

      至此，你已经可以通过 xyz.abc.com 来访问内网网站了，同时外网也可以正常访问，内外网的复制粘贴再也不是痛点，工作效率大大提升。
